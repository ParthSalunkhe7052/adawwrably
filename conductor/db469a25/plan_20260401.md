# Phase 1: Foundation Implementation Plan

## Overview
Initialize the project with Tailwind CSS v4 and shadcn/ui, setting up the core design system tokens (colors, fonts, assets) for the Adawwrably landing page.

## Scope Definition
### In Scope
- Upgrade React to v19 (as per PLAN-1-FOUNDATION.md).
- Install Tailwind CSS v4 and `@tailwindcss/vite` plugin.
- Initialize and configure shadcn/ui.
- Set up custom design tokens in `src/index.css`.
- Configure `index.html` with correct metadata and fonts.
- Prepare `public/` directory with assets (`Logo.png`, `hero-bg.png`).

### Out of Scope
- Implementing UI components (Hero, Cards, etc.) - Phase 2.
- Interactive logic or data fetching.
- Mobile responsiveness polish (beyond basic Tailwind defaults).

## Implementation Phases

### Phase 1: Dependency & Environment Setup
- **Goal**: Upgrade React and install Tailwind v4.
- **Steps**:
  1. [ ] Upgrade React: `npm install react@latest react-dom@latest`.
  2. [ ] Upgrade Types: `npm install -D @types/react@latest @types/react-dom@latest`.
  3. [ ] Install Tailwind v4: `npm install tailwindcss @tailwindcss/vite`.
  4. [ ] Configure `vite.config.ts`: Add `tailwindcss()` plugin.
- **Verification**: `npm run dev` starts without errors.

### Phase 2: shadcn/ui Initialization
- **Goal**: Set up shadcn/ui framework.
- **Steps**:
  1. [ ] Run `npx -y shadcn@latest init`. (Select default options, use `Slate` or `Zinc` as base if prompted, but we will override CSS).
- **Verification**: `components.json` and `src/components/ui` directory created.

### Phase 3: Design System Configuration
- **Goal**: Apply custom design tokens and update shell.
- **Steps**:
  1. [ ] Update `src/index.css`:
     - Import Tailwind v4.
     - Add `@layer base` with custom color variables (`--primary`, `--secondary`, `--glow-red`, etc.).
     - Add `@theme` with custom fonts (`Bebas Neue`, `Inter`, `DM Mono`).
  2. [ ] Update `index.html`:
     - Change title and favicon.
     - Add Google Fonts imports.
     - Ensure `<html class="dark">`.
- **Verification**: Inspect CSS in browser to confirm variables are applied.

### Phase 4: Asset Preparation
- **Goal**: Organize static assets.
- **Steps**:
  1. [ ] Create `public/products` directory.
  2. [ ] Move `image copy.png` to `public/hero-bg.png`.
  3. [ ] Move `Logo.png` to `public/Logo.png`.
- **Verification**: Assets accessible at `/hero-bg.png` and `/Logo.png`.

## Final Verification

- [ ] Create a test component using a Tailwind v4 class (e.g., `text-primary`).
