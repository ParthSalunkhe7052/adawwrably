# Research: Phase 2 - Component Construction

**Date**: 2026-04-01

## 1. Executive Summary
Phase 2 focuses on building the "pixel-perfect" UI components for the Adawwrably storefront. The groundwork (Phase 1) has established the tech stack (React, Tailwind v4, shadcn/ui) and initial design tokens. This research maps the existing codebase and the detailed implementation requirements from `PLAN-2-COMPONENTS.md`.

## 2. Technical Context
- **Existing Implementation**:
  - `src/App.tsx`: A simple placeholder with a pulse animation.
  - `src/index.css`: Contains the foundational design tokens, including color palettes, font families (Bebas Neue, Inter, DM Mono), and glassmorphism variables.
  - `src/components/ui/`: Contains shadcn components (only `button.tsx` exists so far).
- **Target Components**:
  - Hero Section
  - Navbar + Side HUD
  - Mobile Menu
  - Marquee Announcement Strip
  - Product Flip Cards
  - Collections Grid
  - Footer with Social Buttons
  - Page Reveal Animation

## 3. Findings & Analysis
- **Hero Section**: Requires 3 visual layers (BG image, gradient overlays, content). Needs parallax and fade-out on scroll.
- **Navbar**: Needs fixed positioning, glassmorphism on scroll, and mobile responsiveness via a `Sheet` component.
- **Side HUD**: A desktop-only fixed left menu that tracks section intersection.
- **Product Cards**: A custom "Uiverse" flip card interaction with a spinning border animation on the back and product info on the front.
- **Marquee**: Infinite horizontal scroll for announcements.
- **Footer**: Includes custom social buttons with specific brand colors and corner radius logic.

## 4. Technical Constraints
- **Tailwind v4**: The project is using the latest Tailwind version with `@theme` blocks in CSS.
- **Performance**: High focus on GPU-accelerated animations (`transform`, `opacity`).
- **Assets**: MISSING! `hero-bg.png` and `Logo.png` are not in the `public/` directory. Placeholder generation or procurement is required.

## 5. Architecture Documentation
- **Folder Structure**:
  - `src/components/layout/`: Global layout components (Navbar, Footer, SideHUD, MobileMenu).
  - `src/components/sections/`: Page-level sections (Hero, LatestDrops, Collections, Marquee).
  - `src/components/cards/`: Reusable card components (ProductFlipCard, CollectionCard, SocialButton).
  - `src/components/shared/`: Utility UI components (SectionHeading, PageReveal).
- **Hooks**:
  - `useScrollPosition`: Needed for parallax and navbar behavior.
- **Data**:
  - `src/data/products.ts`: Needs mock data for products and collections.
