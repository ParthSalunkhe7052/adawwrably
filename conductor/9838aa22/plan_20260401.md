# Phase 2: Component Construction - Implementation Plan

## Overview
This plan details the construction of all UI components for the Adawwrably storefront, as specified in `PLAN-2-COMPONENTS.md`. We are transforming a blank slate into a "pixel-perfect" reactive UI.

## Scope Definition (CRITICAL)
### In Scope
- Creation of `src/hooks/useScrollPosition.ts`.
- Creation of `src/data/products.ts` (mock data).
- Building all layout components: `Navbar`, `SideHUD`, `Footer`, `MobileMenu`.
- Building all section components: `HeroSection`, `Marquee`, `LatestDrops`, `Collections`.
- Building all card components: `ProductFlipCard`, `CollectionCard`, `SocialButton`.
- Building shared components: `SectionHeading`, `PageReveal`.
- Integration of all components into `src/App.tsx`.
- Adding necessary CSS animations to `src/index.css`.

### Out of Scope (DO NOT TOUCH)
- Actual shop functionality (cart logic, checkout).
- Backend integration.
- Final asset procurement (using placeholders/generated assets for now).

## Current State Analysis
- `src/App.tsx`: Placeholder content only.
- `src/index.css`: Basic Tailwind v4 and shadcn setup.
- `src/components/ui/`: Contains only `button.tsx`.
- Assets: `hero-bg.png` and `Logo.png` are missing.
